# Centient — Features

Running log of shipped features and scaffolding milestones.

## Scaffolding

- **Project scaffold (issue #13):** Next.js 16 App Router project with React 19.2, TypeScript 5.4, Tailwind 4, Prisma 7, and viem 2.x. Established the directory layout (`app/`, `lib/`, `components/`, `types/`, `prisma/`) with placeholder modules, `next.config.ts` set to `output: "standalone"` for Railway, `.env.local.example` template, and `.gitignore` excluding `node_modules/`, `.env.local`, `app/generated/`, and `.next/`.
- **Prisma 7 data model (issue #14):** `User`, `Task`, and `Submission` models in `prisma/schema.prisma` backed by PostgreSQL via `@prisma/adapter-pg`. Wallet-keyed users, UUID tasks with gold-task flags, and submissions keyed by `(walletAddress, taskId)` with payout tracking. Singleton client in `lib/prisma.ts` reuses the same instance across Next.js hot reloads. Initial migration (`init`) creates `users`, `tasks`, `submissions` plus supporting indexes.
- **Local Postgres via Docker (issue #24):** `docker-compose.yml` defines a `postgres:15-alpine` service on port 5432 with credentials matching `.env.local.example`, a named `centient_pgdata` volume for persistence across restarts, and a `pg_isready` healthcheck. One-command local setup: `docker compose up -d`.

## Libraries

- **Shared utilities (issue #15):** `lib/constants.ts` (Celo chain config + cUSD/USDC addresses + `REWARD_CUSD`), `lib/minipay.ts` (client-only `isMiniPay` + `getWalletAddress` via `eth_requestAccounts`), `lib/payout.ts` (server-side `payCUSD`, `waitForTx`, `rewardInWei` via viem + `privateKeyToAccount`), `lib/quality.ts` (15s in-memory rate limiter + reason-spam validator), `types/index.ts` (`PayoutStatus`, `Choice`, `TaskResponse`, `SubmitRequest`, `SubmitResponse`, `MeResponse`). Bumped `tsconfig.json` target to `ES2020` to allow the `100_000n` BigInt literal in `payout.ts`.

## Data

- **Task seed (issue #16):** `prisma/seed.ts` idempotently upserts 100 task pairs — 90 regular (15 general, 15 coding, 10 writing, 15 math, 15 explanation, 10 creative, 10 advice) and 10 gold (4 factually wrong, 2 empty, 2 inappropriate refusal, 2 nonsense) with deterministic string IDs and a 5-A/5-B gold answer balance. Uses `@prisma/adapter-pg`. Runs via `npm run db:seed`; re-runs are safe thanks to `upsert` keyed on `id`.

## Core flows

- **`GET /api/task` (issue #17):** Returns the next un-submitted task for a wallet. Validates the `wallet` query param against `/^0x[a-f0-9]{40}$/`, picks from gold tasks 10% of the time and regular tasks 90%, orders by `createdAt asc`, excludes tasks the wallet has already submitted, and never exposes `isGold` or `goldAnswer` in the response. Returns `{ task: null, message: "No more tasks available" }` when nothing remains.
- **`GET /api/me` (issue #19):** Returns cumulative earnings and submission count for a wallet. Validates the `wallet` query param against `/^0x[a-f0-9]{40}$/`. For unknown wallets returns zeroed stats (no 404). `totalEarnedCUSD` is formatted via `viem`'s `formatUnits(user.totalEarnedWei, 18)`.
- **`POST /api/submit` (issue #18):** Validates body shape, applies the 15-second rate limit and `isSpamReason` check, upserts the user, rejects banned users (403), rejects duplicate `(walletAddress, taskId)` (409), and looks up the task. For gold tasks: a wrong answer records a skipped submission, increments `goldAttempted`, and bans the user when accuracy `< 50%` after `>= 3` attempts; a correct answer increments `goldCorrect` + `goldAttempted` and falls through. Left-bias guard rejects when `> 95%` of the last 20 submissions are the same side. On success, creates a pending submission, calls `payCUSD`, and atomically marks the submission `sent` while incrementing `submissionCount` and `totalEarnedWei`. Failures mark the submission `failed` and return 500. Response: `{ paid, txHash, explorerUrl }` derived from `NEXT_PUBLIC_EXPLORER_URL`.
- **Main page — full screen flow (issue #22):** `app/page.tsx` is a single client component that drives one of seven `Screen` states: `checking`, `not_minipay`, `loading`, `task`, `no_tasks`, `success`, `quality_failed`, `banned`. On mount it gates on `isMiniPay()`, resolves `getWalletAddress()`, then fetches `/api/me` and `/api/task`. Submit posts to `/api/submit` and switches to `success` (auto-advances after 1.5s with a fresh fetch), `quality_failed` (auto-advances after 1.5s), or `banned` (no advance). Sticky header on the task screen carries the wordmark + `EarningsBadge`; ambient blob background only on `not_minipay` and `success` per the brand spec; success screen links to the explorer URL from `NEXT_PUBLIC_EXPLORER_URL`.
- **MiniPay connect resilience (issue #44):** `lib/minipay.ts` adds `connectMiniPay()`, which races `eth_requestAccounts` against a 10-second timeout and throws on rejection / no accounts / missing provider. `app/page.tsx` swaps the mount effect to `connectMiniPay()` and routes failures into a new `wallet_error` screen ("Couldn't connect to MiniPay" + `link_off` icon in the `error-container` family + a "Try again" CTA that reloads). The page no longer hangs on the loading screen when MiniPay denies, errors, or fails to respond.
- **Account sheet (issue #43):** Tapping the header `EarningsBadge` opens `components/AccountSheet.tsx`, a bottom sheet showing the truncated wallet address (`0x1234…abcd`) with a Copy button that writes the full address to the clipboard, the total earned + token symbol in the success-screen balance treatment, the submission count, and a "View on explorer" link to `${EXPLORER_URL}/address/<wallet>`. Esc, scrim tap, and the explicit close affordance all dismiss. Uses `role="dialog"` + `aria-modal="true"`; wallet address rendered in `font-mono`. `app/page.tsx` tracks `submissionCount` from `/api/me` and passes it through.
- **Submit error surfacing + server logs (issue #57):** `components/Toast.tsx` renders a top-center auto-dismissing toast (4s) in `error-container` for errors / `surface-container-highest` for info, with close button, `aria-live` announcements, and a `centient-toast-in` keyframe declared in `app/globals.css`. `app/page.tsx` `handleSubmit` now catches network failures, tolerates non-JSON responses, and maps known error codes (`rate_limited`, `already_submitted`, `invalid_reason`, `invalid_choice`, `invalid_wallet`, `invalid_task`, `invalid_body`, `left_bias_detected`, `task_not_found`, `payout_failed`, `server_error`) to friendly copy before showing the toast — every non-success path now gives the user visible feedback. `app/api/submit/route.ts` routes every error return through a shared `errorResponse(code, status, context)` helper that logs `[submit] <code>` with wallet / task / error context; unexpected throws at any stage now return JSON `{ error: "server_error" }` instead of leaking an HTML 500, and successful gold-task bans log `[submit] banned_wallet` for audit.
- **Landing page + FAQ, min-loading, wallet chip (issue #60):** New `landing` screen sits between `loading` and `task` on fresh MiniPay mounts. `app/page.tsx` races `connectMiniPay() + /api/me` against a 1.5s `MIN_LOADING_MS` floor so the branded `LoadingScreen` is a visible moment, then transitions to `landing`. `components/Landing.tsx` composes the mascot + wordmark + tagline lockup, a `Your Balance` card (running total from `/api/me`, with "Welcome — let's get started." copy when `submissionCount === 0`), `components/Faq.tsx` (accordion of 5 `<details>`/`<summary>` Q&As pulling `REWARD_AMOUNT` + `REWARD_TOKEN_SYMBOL` from env and appending " Sepolia (testnet)" when `NEXT_PUBLIC_CHAIN_ID === "11142220"`), and a `Start Earning` primary CTA that triggers `fetchTask`. `app/globals.css` hides the native `details` marker. `components/WalletChip.tsx` is a truncated-address pill (`0x1234…abcd`) with `account_balance_wallet` icon; the task-screen sticky header now carries both `WalletChip` and `EarningsBadge` side-by-side — both open the existing `AccountSheet`. Truncation moved to `lib/wallet.ts` (`truncateAddress`) and reused by `AccountSheet`.

## Admin

- **Admin dashboard at `/admin` (issue #63):** Password-protected operator tool with three read-only surfaces. Auth is a single DB-backed `AdminUser` seeded as `admin@centient.work` / `GoCent!123` — overridable via `ADMIN_SEED_PASSWORD`, re-seeds leave the row untouched. `lib/admin-rate-limit.ts` caps each IP at 5 failures per 10 minutes and returns 429 on the 6th. Routes live under `app/admin/(protected)/*` so `app/admin/(protected)/layout.tsx` can call `requireAdmin()` once while `app/admin/login/page.tsx` stays public. Pages: **Tasks** (`/admin/tasks`) — `TaskTable` client component renders all seeded tasks with `all / regular / gold` type pills and dynamic category chips; clicking a row expands prompt + both responses (gold answer highlighted) + last 20 submissions. **Wallets** (`/admin/wallets`) — `WalletTable` client component with sortable `Joined / Submissions / Earned / Gold accuracy` columns, truncated wallet links, and an `error-container` "paused" chip for banned wallets. `lib/admin-data.ts` wraps all Prisma reads + the hot wallet balance lookup. UI reuses brand tokens (no ambient blobs, no hardcoded hex). New env vars: `ADMIN_JWT_SECRET` (required, 32+ chars) and optional `ADMIN_SEED_PASSWORD`.

- **Campaigns placeholder page (issue #93):** `app/admin/(protected)/campaigns/page.tsx` closes the post-login 404 for `CUSTOMER` admins (the login route redirects to `/admin/campaigns`, but the page didn't exist before). Renders a "Coming soon" surface gated by `requireAdmin()`, greets the admin by `companyName` when present, and links to issue #94 where the real campaigns dashboard + CSV upload will land.

- **Admin auth + RBAC (issue #80):** Replaced iron-session with stateless JWT auth via `jose`. `lib/admin-auth.ts` provides `signAdminJWT`, `verifyAdminJWT`, `getAdminSession`, `requireAdmin`, `requireRoleForRoute` (for API routes — returns 403 `NextResponse`), and `requireRoleForPage` (for server components — calls `redirect`). JWT uses HS256, 7-day expiry, stored in `admin_session` httpOnly cookie (Secure in prod, `sameSite: lax`). `middleware.ts` intercepts all `/admin/**` and `/api/admin/**` routes, verifies JWT, redirects to `/admin/login` on missing or invalid cookie. Public paths allowlisted: `/admin/login`, `/api/admin/login`, `/api/admin/logout`. `POST /api/admin/login` validates email + bcrypt, signs JWT with `{ sub, email, role, companyName }`, redirects to `/admin`. `POST /api/admin/logout` clears the cookie. `GET /api/admin/auth/me` returns `{ id, email, role, companyName }`. `/admin` redirects server-side based on role. New env var: `ADMIN_JWT_SECRET` (required, 32+ chars).

## Configuration

- **Env-driven chain + reward token:** Chain selection is driven by `NEXT_PUBLIC_CHAIN_ID` (`celo` for `42220`, `celoSepolia` for `11142220`); RPC URL falls back to the chain's default Forno endpoint. Reward token is fully env-driven: `REWARD_TOKEN_ADDRESS` (server), `NEXT_PUBLIC_REWARD_TOKEN_SYMBOL`, `NEXT_PUBLIC_REWARD_TOKEN_DECIMALS`, and `NEXT_PUBLIC_REWARD_AMOUNT` flow through `lib/payout.ts`'s `payReward`/`rewardInWei`, the `/api/me` formatter, and the in-app reward badge / success screen / balance card. `MeResponse` payload renamed `totalEarnedCUSD` → `totalEarned` and now includes `rewardSymbol`. Defaults preserve mainnet cUSD behaviour; flipping to Sepolia USDC is a config-only change.

- **Gold tasks — platform gold set (approach b) (#155):** Gold tasks used for labeler quality control are sourced exclusively from the platform-maintained gold set — `isGold=true AND campaignId IS NULL`. Campaign uploads can never produce gold tasks (the CSV upload route rejects any CSV containing `isGold` or `goldAnswer` columns). The gold ratio is configurable via `GOLD_TASK_RATIO` env var (default `0.1`). `GET /api/task` splits into two branches: the gold branch queries only platform gold tasks, with automatic fallback to non-gold when the platform gold pool is exhausted. Quality ban logic is otherwise unaffected: `/api/submit` still enforces the actual gold-answer checks against `task.isGold` / `goldAnswer` directly, while `lib/quality.ts` continues to provide the rate-limit and spam-reason helpers. This decouples quality control from customer data and keeps the gold ratio stable regardless of campaign task volume.

- **Async CSV task upload (issue #157):** Replaced the synchronous `POST /api/admin/campaigns/[id]/upload` (which held the request open for the full 10k-row cap) with an accept-and-enqueue flow drained by a background worker. `POST .../upload` now validates the file (type/size/gold-columns), runs the cheap `parseCSV` synchronously, persists the raw CSV body in a new `upload_jobs` row (status `queued`), and returns `202 { jobId, totalRows, status }` immediately — no rows are upserted on the request path. `lib/upload-worker.ts` is a long-poll loop (`pnpm worker`) that atomically claims jobs via `UPDATE ... WHERE status='queued' OR (status='processing' AND workerHeartbeatAt < now() - 60s) RETURNING id` (handles stale-processing recovery from worker crashes), then re-parses the raw text and upserts in 500-row chunks with the same 30 s transaction timeout from #145. After each chunk the worker updates `processedRows`/`insertedRows`/`chunksCommitted` and a `workerHeartbeatAt` timestamp so multiple workers would be safe (currently one replica). `GET /api/admin/campaigns/[id]/upload/[jobId]` exposes the live job state to the detail page, which polls every 1.5 s while the job is in `queued`/`processing` and renders a progress bar + running counts in a new `UploadStatusCard`. `POST .../upload/[jobId]/retry` resets a `failed`/`cancelled` job back to `queued` and re-arms the polling card — idempotency comes from the existing `Task.@@unique([campaignId, prompt])` constraint, so retries never duplicate rows. `SIGTERM`/`SIGINT` set a `shouldStop` flag that lets the in-flight chunk commit before the loop exits. Audit log gains `tasks.upload.queued`, `tasks.upload.completed`, `tasks.upload.failed`, and `tasks.upload.retry` actions. No new infra: this is a DB-backed job table drained by a second Railway service running `pnpm worker` — the same shape we want to converge on for #98 (rate-limits) and #99 (payout retry) when those land.
